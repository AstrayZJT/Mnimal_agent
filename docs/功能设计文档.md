# Minimal Agent 功能设计文档

## 1. 项目目标

本项目是一个 Spring Boot 单体应用，结合 LangChain4j 和 LangGraph4j，实现“真实大模型调用 + 图式生成流程 + 条件循环 + 状态持久化 + 简单前端工作台”。

核心目标有三个：

1. 用户提交写作需求后，系统调用真实大模型生成内容。
2. 生成流程不是一次性输出，而是通过 LangGraph4j 按节点执行，并在评分未达标时循环修正。
3. 用户最终只看到可交付的成品，草稿、评分、trace、checkpoint 等内部信息保存在服务端。

---

## 2. 总体架构

### 2.1 模块划分

- `langchain`：封装真实大模型调用
- `generation`：生成工作流、记录保存、文件下载
- `auth`：注册、登录、退出、当前用户
- `web`：页面路由和登录拦截
- `api`：统一异常返回
- `config`：模型、workflow、checkpoint、应用属性
- `static`：简单前端页面和 JS

### 2.2 请求链路

用户从前端提交需求后，整体链路是：

`前端表单 -> /api/generations -> LangGraph4j 图执行 -> DB 保存 -> 文件落盘 -> 前端展示最终稿`

### 3.2 角色分工

AI 在工作流中承担三个角色：

- `draft`：根据主题和要求生成初稿
- `judge`：对稿件打分并给出修正建议
- `finalize`：根据评分和修正结果输出最终稿

### 3.3 提示词策略

提示词统一由后端拼装，不把业务规则散落在前端。

约束包括：

- 只输出正文，不输出说明
- 评审阶段只输出 JSON
- 最终输出必须符合用户要求和评分标准

### 3.4 条件循环

LangGraph4j 图中的 `judge` 节点会根据分数决定下一步：

- 分数达到阈值，进入 `finalize`
- 分数未达标且修正次数未超限，进入 `revise`
- `revise` 完成后再次回到 `judge`

### 3.5 AI 输出边界

对用户可见的只有最终成品文本。

内部信息包括：

- 草稿
- 评分
- 修正建议
- trace
- threadId

这些信息只存服务端，不下发给用户下载文件。

### 4.3 LangGraph4j 工作流

图结构为：

`START -> draft -> judge -> (revise | finalize) -> ... -> END`

当前实现采用官方风格的链式写法：

- `new StateGraph<>(...)`
- `.addNode(..., node_async(...))`
- `.addConditionalEdges(..., edge_async(...), Map.of(...))`
- `.compile(compileConfig)`

### 4.4 节点持久化

workflow checkpoint 根据数据库类型决定实现：

- PostgreSQL：`PostgresSaver`
- 其它情况：`MemorySaver`

这样既能支持正式环境持久化，也能支持本地轻量开发。

## 5. 简单前端设计

### 5.1 页面组成

前端是静态页面，不依赖独立 Node 项目。

页面包括：

- 登录页
- 注册页
- 工作台页

### 5.2 工作台功能

工作台主要提供四类操作：

- 提交生成
- 查看最终成品
- 复制最终文本
- 下载最终文本

### 5.3 前端展示策略

前端只展示：

- 当前生成的最终稿
- 生成时间
- 历史记录预览

不展示：

- 草稿
- 评分
- 修正建议
- trace

### 5.4 前后端交互

前端通过 `fetch` 调用接口：

- `POST /api/auth/register`
- `POST /api/auth/login`
- `POST /api/auth/logout`
- `GET /api/auth/me`
- `POST /api/generations`
- `GET /api/generations`
- `GET /api/generations/{id}`
- `GET /api/generations/{id}/file`

## 6. 用户流程设计

### 6.1 注册/登录

1. 用户打开注册页
2. 填写用户名、密码、昵称
3. 注册成功后自动登录
4. 进入工作台

### 6.2 生成内容

1. 输入主题、受众、语气、补充要求
2. 提交生成
3. 后端启动 LangGraph4j 图
4. AI 先生成初稿
5. AI 评分
6. 不达标则循环修正
7. 达标后输出最终稿
8. 保存数据库和磁盘
9. 前端显示最终成品

### 6.3 查看历史

1. 页面加载时拉取最近记录
2. 点击历史项查看对应最终稿
3. 可复制或下载
