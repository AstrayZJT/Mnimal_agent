# Minimal Agent 项目说明

## 1. 项目定位

这是一个基于 Spring Boot 的单体 Web 项目，前端页面、后端接口、用户系统、内容生成、历史记录和文件落盘都在同一个应用里完成。

项目名虽然带有 LangGraph4j / LangChain4j，但当前代码里的生成核心实际使用的是 `ToyWritingChatModel`，也就是一个本地的示例聊天模型，用来模拟“先生成草稿，再润色成最终稿”的流程，方便本地演示和开发。

## 2. 核心特点

- 单体部署：一个 Spring Boot 应用同时提供页面和 API。
- 前端静态化：`src/main/resources/static` 里直接放 `html/css/js`，不需要单独起 Node 前端。
- 基于 Session 的登录态：登录后把用户 ID 存进 `HttpSession`。
- 用户隔离：每个用户只能看到自己的生成记录和文件。
- 生成结果可追溯：内容会同时写入数据库和本地磁盘。
- 支持下载：每条生成记录都可以下载对应的 Markdown 文件。
- 本地可运行：`local` profile 下使用 H2 文件库，方便不用先搭 PostgreSQL。

## 3. 技术栈

- 后端：Spring Boot 3.5.x
- Web：Spring Web
- 数据访问：Spring Data JPA
- 校验：Spring Validation
- 安全相关：Spring Security Crypto（用于密码加密）
- 数据库：
  - 默认配置：PostgreSQL
  - 本地开发：H2
- 模型层：LangChain4j / LangGraph4j 相关依赖
- 前端：原生 HTML + CSS + JavaScript

## 4. 项目结构

- `src/main/java/com/example/langgraph4jdemo`
  - `LangGraph4jDemoApplication`：应用入口
  - `auth`：用户注册、登录、当前用户、用户实体
  - `generation`：内容生成、历史记录、文件下载、生成记录实体
  - `langchain`：写作助手模型封装
  - `web`：页面路由、拦截器、Session 常量
  - `api`：统一异常返回
  - `config`：应用配置
- `src/main/resources/static`
  - `login.html` / `register.html` / `app.html`
  - `app.js` / `styles.css`
  - `brand.svg`
- `src/main/resources`
  - `application.yml`：默认数据库和模型配置
  - `application-local.yml`：本地 H2 配置

## 5. 页面与路由

### 页面路由

- `/`：已登录跳转到 `/app`，未登录跳转到 `/login`
- `/login`：登录页
- `/register`：注册页
- `/app`：工作台页面

### 静态资源

前端页面不是 React/Vue 项目，而是 Spring Boot 直接返回静态 HTML。
页面里的 `app.js` 通过 `fetch` 调用后端 API 完成登录、生成、刷新历史、复制结果、退出登录等操作。

## 6. 登录与会话流程

### 注册

1. 用户在注册页输入 `username`、`password`、`displayName`
2. 前端调用 `POST /api/auth/register`
3. 后端校验参数
4. `UserAccountService` 检查用户名是否重复
5. 密码通过 `BCryptPasswordEncoder` 加密后入库
6. 注册成功后自动把用户写入 Session，直接进入工作台

### 登录

1. 用户在登录页输入用户名和密码
2. 前端调用 `POST /api/auth/login`
3. 后端查找用户并校验密码
4. 登录成功后把 `CURRENT_USER_ID` 写入 Session
5. 前端跳转到 `/app`

### 退出登录

1. 前端调用 `POST /api/auth/logout`
2. 后端销毁 Session
3. 页面跳回 `/login`

### 当前用户

- `GET /api/auth/me` 用于判断当前是否登录
- 前端页面进入后会先请求这个接口

## 7. 内容生成流程

这是项目的主流程。

### 一次生成的完整链路

1. 用户登录进入工作台
2. 在表单里填写：
   - `topic`
   - `audience`
   - `tone`
   - `notes`
3. 前端调用 `POST /api/generations`
4. `AuthInterceptor` / `CurrentUserService` 确认用户已登录
5. `ContentGenerationService` 组装 prompt
6. `WritingAssistant` 先生成草稿，再润色成最终稿
7. 生成结果写入数据库表 `generated_texts`
8. `TextArchiveService` 把结果写成 Markdown 文件到本地磁盘
9. 返回前端完整记录
10. 前端刷新历史列表，并展示本次结果

### 生成策略

当前代码里不是直接对接真实大模型，而是使用 `ToyWritingChatModel` 做演示：

- 草稿阶段：根据主题生成一个简短草稿
- 润色阶段：对草稿做一次“改写式”输出

这样做的好处是本地启动稳定，不依赖外部 API 也能完整跑通流程。

## 8. 历史记录与文件下载

### 历史列表

- `GET /api/generations`
- 返回当前登录用户最近的生成摘要
- 按创建时间倒序
- 数量由 `app.history-limit` 控制，默认 20 条

### 详情查看

- `GET /api/generations/{id}`
- 返回单条生成记录的完整内容：
  - topic
  - audience
  - tone
  - notes
  - draftText
  - finalText
  - archivePath
  - createdAt
  - downloadUrl

### 文件下载

- `GET /api/generations/{id}/file`
- 从数据库拿到 `archivePath`
- 读取本地 Markdown 文件并返回下载响应

## 9. 数据模型

### `app_users`

用户表，字段主要包括：

- `id`
- `username`
- `passwordHash`
- `displayName`
- `createdAt`

### `generated_texts`

生成记录表，字段主要包括：

- `id`
- `user_id`
- `topic`
- `audience`
- `tone`
- `notes`
- `draftText`
- `finalText`
- `archivePath`
- `createdAt`

### 关系

- 一个用户对应多条生成记录
- 每条生成记录都关联一个用户

## 10. 文件落盘规则

生成内容会被写到 `app.generated-output-dir` 指定的目录，默认是：

```text
generated-output
```

落盘文件按用户分目录，例如：

```text
generated-output/
  alice/
    20260705-153012-12-langgraph4j-demo.md
```

文件内容会包含：

- 用户信息
- 主题
- 语气
- 受众
- 备注
- 草稿
- 最终稿

## 11. 权限与拦截

### Session 键

- `SessionKeys.CURRENT_USER_ID`

### API 拦截

- `WebConfig` 给 `/api/**` 加了 `AuthInterceptor`
- `AuthInterceptor` 会检查 Session 里是否有当前用户
- 没登录时返回 `401` 和 `{"message":"请先登录"}`
- `GET /api/auth/**` 和 `/error` 被排除，不需要先登录

## 12. 异常处理

项目用了统一异常返回 `RestExceptionHandler`，常见情况包括：

- 用户名已存在：`409`
- 登录失败：`401`
- 记录不存在：`404`
- 参数校验失败：`400`
- 其它未预料错误：`500`

前端 `app.js` 会读取接口返回的 `message` 字段展示给用户。

## 13. 配置方式

### 默认配置

`src/main/resources/application.yml`

- 默认数据库：PostgreSQL
- 默认会读取环境变量：
  - `DB_HOST`
  - `DB_PORT`
  - `DB_NAME`
  - `DB_USERNAME`
  - `DB_PASSWORD`
  - `OPENAI_API_KEY`

### 本地开发配置

`src/main/resources/application-local.yml`

- 使用 H2 文件数据库
- 数据文件保存在 `./data/agentdemo-local`

### 本地启动建议

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

## 14. 本地运行方式

### 在 IDEA 里

1. 用 IDEA 打开项目根目录
2. 识别为 Maven 项目
3. JDK 设为 21
4. 运行 `LangGraph4jDemoApplication`
5. 激活 `local` profile

### 访问地址

- `http://localhost:8080/login`
- `http://localhost:8080/register`
- `http://localhost:8080/app`

## 15. 这个项目适合学习什么

- Spring Boot 单体应用结构
- Session 登录态管理
- Spring Data JPA 实体与仓库设计
- 前后端一体化项目组织方式
- 文件落盘与下载
- 统一异常处理
- “生成草稿 -> 润色 -> 存储 -> 展示” 的业务链路

## 16. 一句话总结

这是一个“登录后生成内容、保存到数据库和磁盘、再从历史记录里回看和下载”的 Spring Boot 示例项目，前端页面和后端接口紧密集成，适合拿来学习完整 Web 业务链路。
