# Minimal Agent 项目说明

## 1. 项目定位

这是一个 Spring Boot 单体项目，集成了 LangChain4j 和 LangGraph4j，用来演示“真实大模型调用 + 图式生成流程 + 状态持久化”。

## 2. 核心特点

- 真实模型调用：通过 LangChain4j 接入 OpenAI-compatible 模型
- 图式工作流：`draft -> judge -> revise -> finalize`
- 条件循环：评分未达标时会自动回写修正
- 节点持久化：PostgreSQL 下使用 LangGraph4j checkpoint saver
- 私密保存：草稿、评分、trace 仅保存到服务端，不下发到下载文件
- 前后端一体：页面由 Spring Boot 静态资源直接提供

## 3. 生成流程

1. 用户登录
2. 提交主题、受众、语气和补充要求
3. 后端创建图执行线程
4. `draft` 节点生成初稿
5. `judge` 节点评分并给出修正建议
6. 如果分数未达标且未超出重试次数，进入 `revise` 节点
7. 循环结束后进入 `finalize` 节点
8. 只把最终成品返回给用户
9. 内部草稿、评分、trace 和 checkpoint 留在服务端

## 4. 模块划分

- `auth`：注册、登录、退出、当前用户
- `generation`：生成链路、记录保存、文件下载
- `langchain`：LangChain4j 模型配置
- `web`：页面路由和登录拦截
- `api`：统一异常返回
- `config`：应用和 workflow 配置

## 5. 数据与存储

- 用户表：`app_users`
- 生成记录表：`generated_texts`
- 下载文件：`generated-output/private/<userId>/<recordId>.md`

下载文件只包含最终文本，不包含草稿、评分、建议或其他内部信息。

## 6. 运行方式

### 本地调试

```bash
mvn spring-boot:run -Dspring-boot.run.profiles=local
```

### 需要的环境变量

- `OPENAI_API_KEY`
- `DB_HOST`
- `DB_PORT`
- `DB_NAME`
- `DB_USERNAME`
- `DB_PASSWORD`

### 说明

- `local` profile 使用 H2 作为应用数据库
- 若 datasource 是 PostgreSQL，LangGraph4j checkpoint 会持久化到数据库
- 若不是 PostgreSQL，checkpoint 会退回到内存保存

## 7. 一句话总结

这是一个能把“真实模型 + 图工作流 + 条件循环 + 私密保存”串起来的写作代理示例项目。
